Arbetsflöde

Automatisering av HMTL, CSS och javascript

Denna uppgift har automatisering/optimering av filer blivit utfört genom en mjukvara vid namn Parcel.

Utan en automering behöver all form av optimering av filer och bilder utföras manuellt. Det som parcel blandannat utför, är att sammanstråla alla filer till en inom varje del i en hemsida.

Tex om ett projekt består av 3 stycken javascript filer, så sammanslås dessa filer till en. desamma gäller för CSS. Till Parcel finns det flera plugin/paket som kan användas, dessa kan tex vara för bildoptimering som sharp.

För att aktivera Parcel kör man följande kommando i kommandotolken:

npm start

För att komma igång startar arbetet med att konfigurera en Package-fil som är baserad på Json. Denna fil är det som styr vilka devDependencies som behövs och installeras när parcel kör. Detta innebär att man ej behöver kopiera paket som ett projekt bygger på. 

    
    {
    "name": "moment1",
    "version": "1.0.0",
    "description": "Packagefil for moment 1",
    "source": "src/index.html",
    "scripts": {
        "start": "parcel",
        "build": "parcel build"
    },
    "author": "Niklas Borg",
    "license": "ISC",
    "devDependencies": {
        "parcel": "^2.11.0",
        "sharp": "^0.31.3"
    }
    }

Denna Package fil underlättar installation av alla devDependencies som behövs för att kunna kompilera/optimera filer i hemsidan. Fillen skapas genom att köra följande kommando:

npm instal parcel --save-dev
